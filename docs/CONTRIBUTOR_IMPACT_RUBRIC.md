# Contributor Impact Rubric

This document explains how the CML contributor impact board should be interpreted.

The goal is recognition, not competition.

## Purpose

The contributor impact board highlights concrete project value created by community members.

It helps show:

- who improved reproducibility;
- who improved documentation clarity;
- who improved research framing;
- who improved tests or validation;
- who helped reviewers understand the project faster.

## What the score means

The impact score is a qualitative maintainer signal.

It is not:

- a formal ranking;
- a payment score;
- a leaderboard of personal worth;
- a claim that one contributor is better than another;
- a permanent score.

It is a lightweight way to communicate visible project impact at a specific moment in the repository's development.

## Scoring dimensions

Scores should consider several dimensions.

| Dimension | Question |
| --- | --- |
| Reproducibility | Did the contribution make CML easier to run or verify? |
| Research value | Did it clarify a failure class, taxonomy, or future benchmark path? |
| Onboarding value | Did it help new contributors or reviewers understand the project? |
| Trust value | Did it improve external validation, tests, CI, or evidence quality? |
| Strategic value | Did it help explain where CML might matter in real systems? |
| Merge status | Was the contribution merged, manually applied, superseded, or still waiting? |

## Suggested interpretation bands

| Band | Meaning |
| --- | --- |
| 90-100 | Major project-shaping contribution across multiple dimensions. |
| 70-89 | Strong merged contribution with clear reviewer or contributor value. |
| 50-69 | Useful contribution, early direction, or partially superseded work. |
| 30-49 | Helpful attempt or in-progress work that still needs fixes or validation. |
| 0-29 | Reserved for experimental, stale, or not-yet-actionable work. |

## How to avoid harm

Maintainers should avoid using the board to shame, pressure, or compare people aggressively.

Good framing:

```text
This contribution improved CML's reproducibility path.
```

Bad framing:

```text
This contributor is better than another contributor.
```

The board should always favor gratitude, specificity, and context.

## Update policy

Impact scores may change when:

- a PR is merged;
- a PR is superseded;
- a contribution is manually applied;
- a follow-up contribution increases the value of earlier work;
- a stale contribution becomes active again;
- a maintainer wants to make the board more accurate or fair.

## Opt-out principle

If a contributor does not want to appear on the public impact board, the maintainer should remove or anonymize the entry.

## Bottom line

The impact board exists to make community value visible.

It should help contributors feel seen and help reviewers understand that CML is becoming a living open-source research project.
